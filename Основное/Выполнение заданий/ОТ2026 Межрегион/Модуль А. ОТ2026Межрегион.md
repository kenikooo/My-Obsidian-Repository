
### 3. Установка и настройка LDAP-Сервера

```bash
apt-get update && apt-get install haveged chrony tzdata zip freeipa-server-dns
```
Запускаем демон-ускоритель и службу времени:
```bash
systemctl enable --now {haveged,chronyd}
```
Меняем временную зону (на первый раз выдает ошибку, поэтому используем ее два раза):
```bash
timedatectl set-timezone Europe/Moscow
```
[Настройка FreeIPA](https://docs.altlinux.org/ru-RU/alt-server/11.1/html/alt-server/freeipa--chapter.html)

Задаем имя в формате FQDN:
```bash
hostnamectl set-hostname infra-srv1.au-team.cloud;exec bash
```
Дополняем /etc/hosts:
```bash
192.168.0.103 infra-srv1.au-team.cloud infra-srv1
```
*требуется для гарантированного разрешения имени хоста*
После чего, запускаем автоматическую установку:
```bash
ipa-server-install -U --hostname=$(hostname) \
-r AU-TEAM.CLOUD -n au-team.cloud -p P@ssw0rd -a P@ssw0rd \
--setup-dns --forwarder 77.88.8.8 --auto-reverse
```
Когда инициализация будет успешной, получим права Kerberos и создадим требуемых пользователей, группы, после этого раскидаем пользователей по группам:
```bash
# Получаем права администратора
kinit admin
# Создаем пользователей
echo "P@ssw0rd" | ipa user-add admin01 --first=admin --last=01 --password
echo "P@ssw0rd" | ipa user-add dev01 --first=dev --last=01 --password
# Создаем группы
ipa group-add developers
# группа admins уже существует в freeipa, поэтому создана командой не будет
# Добавляем пользователей в группу
ipa group-add-member admins --users=admin01
ipa group-add-member developers --users=dev01
```
Присоединим клиентов (команды выполняются на infra-srv2 и infra-srv3, в дальнейшем и на k8s-srv1..5)
```bash
hostnamectl set-hostname ИМЯ_МАШИНЫ.au-team.cloud;exec bash
```
Установим требуемые пакеты:
```bash
apt-get update && apt-get install freeipa-client chrony zip
```
Запустим службу времени:
```bash
systemctl enable --now chronyd
```
Меняем временную зону (выполняем команду два раза, на первый раз будет FAILED, на второй - команда пройдет):
```bash
timedatectl set-timezone Europe/Moscow
```
Присоединим клиента к домену:
```bash
ipa-client-install -U -p admin -w P@ssw0rd
```
На всех устройствах необходимо прописать следующее:
```bash
control sudo public
```

### 4. Установка и настройка сервера Git

Все последующие команды выполняются на `infra-srv2`.
Устанавливаем необходимые пакеты:
```bash
apt-get update && apt-get install gitea MySQL-server
```
После установки, запустим и настроим базу данных:
```bash
systemctl enalbe --now mysqld
```
Открываем mysql:
```mysql
[root@infra-srv2 ~] mysql
mysql: CREATE USER 'giteauser'@'localhost' IDENTIFIED BY 'P@ssw0rd';
mysql: CREATE DATABASE giteadb;
mysql: GRANT ALL PRIVILEGES ON giteadb.* TO 'giteauser'@'localhost' WITH GRANT OPTION;
mysql: FLUSH PRIVILEGES;
```
Переводим MySQL в режим сервера:
```bash
control mysqld server
systemctl restart mysqld
```
Переходим в редактирование `app.ini` для доступа к gitea с любой ВМ: 
```ini
HTTP_ADDR = 127.0.0.1
# Меняем на:
HTTP_ADDR = 0.0.0.0
```
Запускаем службу `gitea`:
```bash
systemctl enable --now gitea
```

Также создадим папку `group_vars` и в нем опишем файл `all.yaml`:
```yaml
---
ansible_ssh_user: altlinux
ansible_ssh_private_key_file: ~/.ssh/id_rsa
ansible_python_interpreter: /usr/bin/python3
```
Приступим к написанию `ansible-playbook`:
`playbook1_base_package.yml`:
```bash
---
- name: Установка пакетов
  hosts.: all
  become: yes
  tasks:
    - name: Установка всего
      community.general.apt_rpm:
        name: "{{ item }}"
        state: present
      with_items:
      - bash-completion
      - vim-console
      - chrony
      - wget
      - curl
      - tzdata
      - apt-repo
```
`playbook2_enter_in_domain.yml`:
```bash
---
- name: Присоединение к домену freeipa
  hosts: all
  become: yes
  tasks:
  - name: Установка всего
      community.general.apt_rpm:
        name: "{{ item }}"
        state: present
      with_items:
      - freeipa-client
      - zip
        
  - name: Смена временной зоны
    community.general.timezone:
      name: Europe/Moscow
      
  - name: Запуск службы времени
    ansible.builtin.systemd:
      name: chronyd
      state: started
      enabled: yes
      
  - name: Подгон имен в FQDN формат
    ansible.builtin.hostname:
      name: {{ ansible_facts['fqdn'] }}
      
  - name: Присоединение к домену
    ansible.builtin.command:
      cmd: "ipa-client-install -U -p admin -w {{ freeipa_admin_password }}"
      creates: /etc/ipa/default.conf
    register: joindomain
    
  - name: Перезапуск машин
    ansible.builtin.reboot:
	  reboot_timeout: 3600
    when: joindomain.changed
```
### 8. Установка и настройка системы мониторинга

Создаем рабочий каталог, на `infra-srv3`, под `root`:
```bash
mkdir -p /opt/monitoring/{grafana,prometheus}
cd /opt/monitoring
```
Описываем конфигурационный файл:
```bash
services:
	prometheus:
		image: prom/prometheus:v3.9.0
		container_name: prometheus
		volumes:
		  - ./prometheus/prometheus.yml:/etc/prometheus/prometheus.yml
		  - prometheusdata:/prometheus
		ports:
		  - "9090:9090"
	grafana:
		image: grafana/grafana:main
		container_name: grafana
		volumes:
		  - ./grafana/datasources.yaml:/etc/grafana/provisioning/datasources.yaml
		  - grafanadata:/var/lib/grafana
		environment:
		  - GF_SECURITY_ADMIN_USER=admin
		  - GF_SECURITY_ADMIN_PASSWORD=P@ssw0rd
		ports:
		  - "3000:3000"
volumes:
  prometheusdata:
  grafanadata:
```
`./grafana/datasources.yaml`:
```bash
apiVersion: 1

datasources:
	- name: Prometheus
	  type: prometheus
	  url: http://prometheus:9090
	  access: proxy
	  isDefault: true
	  editable: true
```
`./prometheus/prometheus.yml`:
```bash
global:
  scrape_interval: 5s
  evaluation_interval: 5s
  
scrape_configs:
	- job_name: "node"
	  static_configs:
		  - targets:
		    - infra-srv1.au-team.cloud:9100
		    - infra-srv2.au-team.cloud:9100
		    - infra-srv3.au-team.cloud:9100
		    labels:
			    target_type: production
		  - targets:
		    - k8s-srv1.au-team.cloud:9100
		    - k8s-srv2.au-team.cloud:9100
		    - k8s-srv3.au-team.cloud:9100
		    - k8s-srv4.au-team.cloud:9100
		    - k8s-srv5.au-team.cloud:9100
		    labels:
			    target_type: staging
		      
```
Создание корневого сертификата
```bash
openssl req -x509 -new -nodes -newkey rsa:2048 -keyout rootCA.key -sha256 -days 365 -out rootCA.crt
```
grafana.ext, содержимое:
```bash
authorityKeyIdentifier=keyid,issuer
basicConstraints=CA:FALSE
subjectAltName = @alt_names
[alt_names]
DNS.1 = grafana.au-team.cloud
IP.1 = 192.168.0.103
```
Выпуск `.csr`:
```bash
openssl req -new -newkey rsa:2048 -keyout grafana.key -out grafana.csr -nodes
```
Подписание сертификата `grafana.crt`:
```bash
openssl x509 -req -in grafana.csr -CA root.ca.crt -CAkey rootCA.key -CAcreateserial -out grafana.crt -days 365 -sha256 -extfile grafana.ext
```
