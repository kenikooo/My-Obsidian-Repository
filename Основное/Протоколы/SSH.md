#putty #ssh
Для безопасного подключения к облачной виртуальной машине (ВМ) используется метод аутентификации с помощью пары криптографических ключей: открытого (public key) и закрытого (private key).
Для создания пары ключей используется специальная программа, одна из таких, PuTTYgen.
* Приватный ключ (private key), требуется сохранить в расширении `*.pem` - стандартный открытый формат, используемый в OpenSSL и в большинстве облачных провайдеров.
Для того, чтобы изменить расширение: переходим к private.ppk - и просто переименовываем `private.ppk` -> `private.pem`. Если вы не видите расширения, в проводнике, перейдите в "Настройки" -> "Вид" и в "Дополнительные параметры", уберите галочку "Скрывать расширения для зарегистрированных типов файлов".
После того, как файл переименован, переходим в PuTTy -> Connection -> SSH -> Auth -> Credentials, и в строке `Private key file for authentication` подставляем наш private.pem:
![[Pasted image 20260204111615.png]]
Возвращаемся в Session, вписываем параметры для подключения в формате `пользователь@адрес` - `altlinux@10.0.0.50`. В качестве подключения используем SSH. Для удобства, сессию можно сохранить (раздел Saved Sessions), чтобы при последующих подключениях подключаться в один клик.

### Ошибка WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED! И ее решение
Если по какой-то из причин, вылезла данная ошибка, к примеру, при подключении на удаленное облачное устройство:
```bash
ssh altlinux@172.16.1.4
```
Исправь ее, можно с компьютера, которого происходит удаленное подключение. Для этого, используем следующую команду:
```bash
ssh-keygen -R 172.16.1.4
```
После этого, подключение будет успешным!
